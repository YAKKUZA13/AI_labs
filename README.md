## Лабораторная работа 1

Тема: загрузка, очистка и анализ пропусков в данных.

Используется набор данных о клиентах телекоммуникационной компании. Целевая переменная — `churn`, то есть факт оттока клиента.

Файл `lab_1/gen_task01_telecom_churn.py` добавлен для соответствия названию генератора из условия варианта 1. Файл `lab_1/generate_data.py` используется как основной нормализованный генератор в общем порядке запуска.

Основные столбцы датасета:

- `customer_id` — идентификатор клиента;
- `age` — возраст клиента;
- `gender` — пол клиента;
- `region` — регион обслуживания;
- `tenure_months` — срок обслуживания в месяцах;
- `monthly_charges` — ежемесячный платёж;
- `num_services` — количество подключённых услуг;
- `support_calls` — количество обращений в поддержку;
- `satisfaction_score` — оценка удовлетворённости;
- `contract_type` — тип договора;
- `payment_method` — способ оплаты;
- `churn` — целевая переменная.

Результаты:

- `lab_1/artifacts/customer_churn.csv`
- `lab_1/artifacts/data_after_step3.csv`
- 7 диагностических графиков в формате PNG

Пояснительная записка:

- `lab_1/report.md`

## Лабораторная работа 3

Тема: разделение данных, нормализация и финальная проверка.

В работе выполняются:

## Проверка результатов

Скрипты содержат проверки корректности обработки данных:

- лабораторная 0 проверяет 10 предложений и сохраняет точность распознавания языка;
- лабораторная 1 проверяет отсутствие пропусков и дубликатов по `customer_id`;
- лабораторная 2 проверяет отсутствие пропусков, бесконечных значений и строковых столбцов;
- лабораторная 3 проверяет одинаковый набор признаков в train/test, стратификацию и корректное применение `StandardScaler` только к обучающей выборке.
