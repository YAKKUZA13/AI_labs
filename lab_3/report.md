# Лабораторная работа 3

## Титульный лист

Дисциплина: Системы искусственного интеллекта.

Тема: Разделение данных, стандартизация и финальная проверка.

Вариант: 1, отток клиентов телекоммуникационной компании.

Номер лабораторной работы: 3.

## Цель работы

Разделить подготовленный датасет на обучающую и тестовую выборки, выполнить стандартизацию числовых признаков без утечки данных, проверить итоговые данные и сохранить файлы, готовые для дальнейшего обучения модели.

## Исходные данные

Входной файл: `../lab_2/artifacts/data_after_step6.csv`.

Этот файл является результатом лабораторной работы 2 и содержит только числовые столбцы.

## Ход выполнения

### Шаг 7. Разделение на обучающую и тестовую выборки

Целевая переменная — `churn`. Идентификатор `customer_id` исключается из матрицы признаков, так как он не является содержательным предиктором.

Данные разделяются следующим образом:

- 80% данных — обучающая выборка;
- 20% данных — тестовая выборка;
- `random_state=42` используется для воспроизводимости;
- стратификация по `churn` сохраняет баланс классов.

Сформирован график:

- `artifacts/step7_split.png`.

### Шаг 8. Стандартизация признаков

Непрерывные числовые признаки стандартизируются с помощью `StandardScaler`.

Масштабировщик обучается только на `X_train`, после чего рассчитанные по обучающей выборке статистики применяются к `X_test`. Бинарные и one-hot признаки не нормализуются, так как они уже находятся в масштабе 0/1. Такой порядок предотвращает утечку данных, потому что тестовая выборка не влияет на параметры преобразования.

Сформированы графики:

- `artifacts/step8_normalization_compare.png`;
- `artifacts/step8_boxplots_normalized.png`.

### Шаг 9. Финальная проверка и сохранение

Скрипт проверяет, что:

- обучающая и тестовая выборки имеют одинаковый набор признаков;
- в данных нет пропусков;
- в данных нет бесконечных значений;
- баланс классов сохранён после стратифицированного разделения;
- стандартизированные признаки обучающей выборки центрированы после обработки;
- идентификаторы строк, список признаков, параметры разделения и параметры масштабировщика сохранены для воспроизводимости.

Сформированы графики:

- `artifacts/step9_final_correlation.png`;
- `artifacts/step9_top_features.png`.

## Результаты тестирования

Итоговые файлы сохраняются в папку `artifacts/`:

- `X_train.csv`;
- `X_test.csv`;
- `y_train.csv`;
- `y_test.csv`;
- `train_ids.csv`;
- `test_ids.csv`;
- `preprocessing_metadata.json`.

Файл `preprocessing_metadata.json` содержит порядок признаков, список масштабированных и немасштабированных столбцов, параметры разбиения и статистики `StandardScaler`, что позволяет воспроизвести преобразования.

## Анализ эффективности выбранных методов

Разделение 80/20 оставляет достаточный объём данных для обучения и одновременно выделяет отдельную тестовую часть для дальнейшей оценки качества модели.

Стратификация важна, потому что `churn` является бинарной целевой переменной, а соотношение классов должно сохраняться в обеих выборках.

Стандартизация необходима, так как непрерывные признаки имеют разные единицы измерения и диапазоны: возраст, срок обслуживания, платежи, количество услуг и инженерные признаки находятся в разных масштабах. Бинарные признаки оставлены без масштабирования, что соответствует требованиям задания.
