# Лабораторная работа 1

## Титульный лист

Дисциплина: Системы искусственного интеллекта.

Тема: Загрузка, очистка и анализ пропусков в данных.

Вариант: 1, отток клиентов телекоммуникационной компании.

Номер лабораторной работы: 1.

## Цель работы

Загрузить исходный набор данных, изучить его структуру, удалить дублирующиеся записи клиентов, проанализировать пропуски и подготовить очищенный набор данных для следующей лабораторной работы.

## Исходные данные

Данные формируются скриптом `generate_data.py` и сохраняются в файл `artifacts/customer_churn.csv`.

Набор данных содержит признаки клиентов телекоммуникационной компании:

- демографические признаки: `age`, `gender`, `region`;
- признаки использования услуг: `tenure_months`, `num_services`, `support_calls`;
- финансовые признаки: `monthly_charges`, `payment_method`, `contract_type`;
- показатель качества обслуживания: `satisfaction_score`;
- целевая переменная: `churn`.

Генератор намеренно добавляет пропуски, дубликаты клиентов и выбросы, чтобы данные подходили для отработки этапов предварительной обработки.

## Ход выполнения

### Шаг 1. Загрузка и первичный осмотр

Скрипт загружает CSV-файл с помощью `pandas.read_csv`, выводит размер датасета, первые строки, типы столбцов, количество пропусков и описательную статистику.

Сформированы графики:

- `artifacts/step1_hist_numeric_before.png`;
- `artifacts/step1_categorical.png`.

Эти графики показывают распределения числовых признаков и частоты категорий до очистки данных.

### Шаг 2. Поиск и удаление дубликатов

Дубликаты проверяются двумя способами:

- полные дубликаты строк;
- совпадения по идентификатору клиента `customer_id`.

Основной критерий удаления дубликатов — поле `customer_id`, так как один идентификатор должен соответствовать одной записи клиента. Повторные записи удаляются с сохранением первого вхождения.

Сформирован график:

- `artifacts/step2_duplicates.png`.

### Шаг 3. Анализ и обработка пропусков

Пропуски визуализируются до и после обработки. Для числовых признаков используется KNN-импутация: значение восстанавливается по похожим строкам, а не заменяется одной общей статистикой.

Категориальные признаки заполняются модой:

- `payment_method` заполняется наиболее частым значением внутри группы `contract_type`;
- `region` заполняется наиболее частым значением внутри группы `payment_method`;
- оставшиеся пропуски заполняются общей модой соответствующего столбца.

Сформированы графики:

- `artifacts/step3_missing_heatmap_before.png`;
- `artifacts/step3_missing_bars.png`;
- `artifacts/step3_missing_heatmap_after.png`;
- `artifacts/step3_distributions_compare.png`.

## Код решения

Код решения находится в файлах:

- `gen_task01_telecom_churn.py`;
- `generate_data.py`;
- `lab1_steps_1_3.py`.

Эти `.py` файлы приложены в папке лабораторной работы и могут быть запущены для воспроизведения результатов.

## Результаты тестирования

В скрипте предусмотрены проверки:

- после обработки не должно оставаться дубликатов по `customer_id`;
- после обработки не должно оставаться пропусков;
- количество строк после удаления дубликатов должно быть меньше исходного количества строк.

Итоговый файл сохраняется как `artifacts/data_after_step3.csv`.

## Анализ эффективности выбранных методов

Удаление дубликатов по `customer_id` предотвращает искусственное увеличение веса отдельных клиентов при дальнейшем анализе.

KNN-импутация выбрана для числовых признаков, потому что показатели клиентов связаны между собой: срок обслуживания, платежи, обращения в поддержку и удовлетворённость могут помогать более точно восстанавливать пропущенные значения.

Заполнение категориальных признаков групповой модой выбрано потому, что способ оплаты и регион могут иметь разные частоты в разных группах клиентов.

## Вывод

В ходе лабораторной работы выполнена загрузка данных, проведён первичный анализ, удалены дубликаты клиентов и обработаны пропущенные значения. Результирующий датасет не содержит пропусков и дубликатов по `customer_id`, поэтому может использоваться как входной файл для лабораторной работы 2.
