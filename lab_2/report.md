## Ход выполнения

### Шаг 4. Обработка выбросов

Сначала применяются предметные ограничения для телекоммуникационных данных:

- `age` должен находиться в реалистичном диапазоне возраста клиента;
- `monthly_charges` не может быть отрицательным;
- `num_services` не может быть отрицательным или нереалистично большим;
- `satisfaction_score` должен находиться в диапазоне от 1 до 5;
- `support_calls` не может быть отрицательным;
- `tenure_months` должен быть положительным.

Физически невозможные значения заменяются на `NaN`, затем заполняются медианой. После этого для оставшихся экстремальных значений применяется ограничение по межквартильному размаху.

- `artifacts/step4_boxplots_after.png`;

### Шаг 5. Кодирование категориальных признаков

Категориальные признаки переводятся в числовой вид:

- `gender` кодируется бинарным признаком `gender_male`;
- `contract_type` кодируется порядковым признаком, так как срок договора имеет естественный порядок;
- `region` и `payment_method` кодируются методом one-hot encoding с удалением базовой категории (`drop_first=True`).

## Результаты тестирования

В скрипте предусмотрены проверки:

- входные данные не должны содержать пропуски после лабораторной работы 1;
- во входных данных не должно быть дубликатов по `customer_id`;
- после обработки не должно оставаться пропусков;
- после создания признаков не должно быть бесконечных значений;
- после кодирования не должно оставаться строковых столбцов.

Итоговый файл сохраняется как `artifacts/data_after_step6.csv`.

## Анализ эффективности выбранных методов

Предметные правила применяются до статистической обработки, потому что невозможные значения являются ошибками данных, а не редкими, но допустимыми наблюдениями.

Ограничение по IQR используется после исправления очевидных ошибок: оно снижает влияние экстремальных значений, но сохраняет строки в наборе данных.

Комбинация бинарного, порядкового и one-hot кодирования выбрана из-за различной природы категориальных признаков. Для пола достаточно бинарного признака, для длительности договора уместен порядок, а для региона и способа оплаты нельзя вводить ложную иерархию категорий.

Новые признаки добавляют интерпретируемые показатели, которые могут быть связаны с оттоком: длительный срок обслуживания, большое число обращений в поддержку, низкая удовлетворённость и высокая стоимость услуги на один подключённый сервис.
